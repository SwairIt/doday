## Бэклог — отложено до решения пользователя

- [ ] **Авто-синк школьной домашки через браузерное расширение / userscript**

  Когда пользователь решит — сказать в чате «делай авто-синк школы» и я
  возьмусь. До этого момента **не делать сам**.

  **Зачем:** заменить ручной paste-import. Сейчас юзер копирует JSON из портала
  и вставляет в Doday. Хочется чтобы дз появлялись в Doday автоматически
  при заходе на школьный портал.

  **Какой вариант выбран:** Вариант 1 (полу-автомат, клиент-side). НЕ серверный
  бот — серверный путь отброшен из-за ToS-рисков, 152-ФЗ и зависимости от
  чужой инфраструктуры (см. историю чата 2026-05-08, обсуждение «можно ли с
  json парсить»).

  **Что нужно сделать:**
  1. Userscript для Tampermonkey (или extension `.zip` для Chrome) который:
     - Срабатывает на `*.mosreg.ru` и `*.mos.ru` (школьные домены)
     - Читает JSON со страницы (домашка возвращается с бэка как JSON в
       Network-вкладке — выясни точные эндпоинты при разработке)
     - Сравнивает с локальным state (`localStorage`: `doday-school-last-sync`)
     - Новые/изменённые задания шлёт `POST /api/school/import` (эндпоинт
       уже существует для paste-import — может потребовать минорной правки
       чтобы принимать инкремент, не полную перевыгрузку)
  2. На `/app/profile` в школьной секции — инструкция «Установи Tampermonkey
     → жми «Скачать userscript» → дальше дз будут синкаться сами при заходе
     на портал».
  3. Endpoint `GET /api/school/userscript.user.js` — отдаёт сам скрипт с
     embedded токеном пользователя для аутентификации в Doday API.
  4. Логи: на стороне Doday логировать каждый автосинк (timestamp, кол-во
     добавленных задач), чтобы юзер видел в Профиле «последний синк: 2 мин назад,
     +3 задачи».

  **Легальная аргументация (заранее, чтобы потом не разбираться):**
  - Это сессия пользователя в его браузере с его логином → читаем его данные
  - Сервер портала видит обычный визит, никакой подозрительной активности
  - Юзер явно установил скрипт → дал согласие
  - Аналогично Notion Web Clipper / Save-Tab-As-Markdown
  - НЕ нарушает ToS «no scraping» потому что не scraping — это user-agent,
    дёргающий API в рамках обычной user-сессии

  **Время реализации:** 3-4 часа. Не Phase 1 — отложено до Phase 2 (мягкий
  запуск, после первых 5-10 юзеров — посмотреть нужна ли вообще, или
  paste-import достаточно).
