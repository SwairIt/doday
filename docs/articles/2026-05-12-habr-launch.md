# Мне 15, и я за 10 дней склепал свой todo с веб-апом, Telegram Mini App и ботом. Открытый код, без подписок и без AI-вранья

**TL;DR:** Привет, Хабр. Меня зовут Ярослав, мне 15. С 2 по 12 мая я в режиме нон-стоп писал [Doday](https://getdoday.ru) — кросс-платформенный todo: web, Telegram Mini App, чат-бот. Получилось 317 коммитов, ~40 000 строк кода, 633 теста, 38 модулей в `app/`. Стек — FastAPI + HTMX + Alpine.js + Tailwind, без единой строчки React. Писал в паре с Claude Code, и не скрываю этого — расскажу как именно и какая часть кода реально моя голова, а какая — клавиатура. Грабли, цифры, ссылки внутри.

> 🚀 Сайт: [getdoday.ru](https://getdoday.ru) · бот: [@DodayTaskBot](https://t.me/DodayTaskBot) · код: [github.com/SwairIt/doday](https://github.com/SwairIt/doday)

## Зачем оно вообще

Я учусь в школе. Дневник у нас — [Школьный портал МО](https://school.mosreg.ru), у соседних классов — МЭШ. UX обоих — это отдельная боль: чтобы посмотреть домашку на завтра, надо кликнуть в три уровня, дождаться спиннера, проскроллить, дождаться второй спиннер, и в итоге увидеть «прочитайте §15». Никаких напоминаний. Никаких приоритетов. Никакой синхронизации между устройствами кроме «открыть сайт в браузере».

Я пользовался Todoist — нравится, но у него есть проблемы: жадная подписка (Premium 229₽/мес), нет интеграции с дневниками, никакой геймификации (а в 15 это работает). И главное — он не *мой*.

2 мая написал первый коммит: «SchoolTodo — туду-лист для школьников». 3 мая, через сутки, понял что концепция «только для школьников» обрезает аудиторию по тупой причине: список дел нужен всем. **Pivot на следующий день после старта.** Брендирую `Doday`, оставляю парсинг дневников как опциональную фичу, делаю аудиенс-режимы (`school` / `company` / `personal`) — стартовые задачи и UI меняются под выбранный.

Через 10 дней — то, что есть сейчас.

## Что получилось

### Web-апа

- Задачи: приоритеты P1-P4, дедлайны, повторения (день/неделя/месяц/год), описания, лейблы, проекты, секции внутри проектов, пин, привычки, ссылки на внешние ресурсы.
- Inbox + Today + Upcoming + Calendar (месяц/неделя) + Projects (list + kanban) + Stats + Profile.
- [Quick-add парсер](https://github.com/SwairIt/doday/blob/master/app/quickadd/parser.py): пишешь `Купить молоко завтра !!! @дом #хозяйство` — задача, дата `завтра`, priority `!! = P2`, лейбл `@дом`, проект `#хозяйство`. Грамматика на стейт-машине, парсит русские даты «через 3 дня», «в пятницу», «15 декабря».
- Поиск, фильтры, шаблоны задач (повторяющиеся типа «учебная неделя»).
- Pomodoro с серверной БД-сессией (25/5, конфигурируемо).
- Геймификация: XP за выполнение, уровни, 16 ачивок («первая задача», «неделя без пропуска», «100 задач»), ежедневные челленджи.
- Mood-tracker, time-tracking, trash-bin, undo-toast.
- Audience-режим: школа → виджет «Расписание» + предметные чипы; компания → «Stand-up Today»; личное → классический.
- Pricing-страница пока показывает «🚀 Сейчас всё бесплатно. Ранние юзеры останутся на Pro».

### Telegram Mini App

Это та часть, которой я больше всего гордая (и которая по-человечески нерешаемая без долгих ночных дебагов — см. ниже).

Открывается через кнопку в чате `@DodayTaskBot` → меню → `Doday`, или через `/app`. Внутри — почти всё то же что в вебе, в нативе и в темной теме по умолчанию:

- 5 табов снизу: Today / Inbox / Calendar / Projects / Профиль.
- Свайп задачи вправо — «Готово», влево — «Завтра» с реальной spring-анимацией (`cubic-bezier(0.34, 1.56, 0.64, 1)`).
- Long-press на задаче — bottom-sheet quick-actions, multi-select bulk-mode.
- Drag-to-reorder с handle-захватом, drag между секциями в kanban.
- Pomodoro mini-widget — плавает над списком, разворачивается в fullscreen с анимацией.
- Confetti при 100%-completion дня (с цветами под priority задачи).
- Skeleton-shimmer на загрузке, staggered fade-in для списков, hero-blob с CSS @keyframes blob-drift.
- 5 SVG-иллюстраций нарисовал руками для empty-states (никаких stock-имиджей).
- Переключатель темы: тёмная / светлая / системная, хранится в localStorage, синхронизирует `setHeaderColor` Telegram'у чтобы chrome-бар поверх Mini App тоже перекрашивался.

### Telegram-точка входа

`@DodayTaskBot` — это не «бот с командами», а **точка запуска Mini App**:

- В чате с ботом снизу — кнопка «Doday», тап → открывается Mini App.
- Описание бота на странице — короткий pitch + кнопка «Запустить».

Чат-команды (`/add`, `/today`, утренний дайджест, напоминания) **в работе** — на текущем хостинге провайдер блокирует исходящие соединения к 2 из 3 DC Telegram'a, polling не выживает. Делаю webhook-вариант отдельно после статьи — там Telegram сам POSTит к нам, обход и DNS и blocked DC. Пока всё через Mini App.

## Стек: почему не React

Я учу JS медленнее чем Python — в школе у нас Pascal/C++ в основном. Решение учить React в момент когда нужно «как можно быстрее довезти что-то рабочее» — путь к brain-freeze. Поэтому стек выбирал по принципу «**максимум фичей при минимуме JS-боли**».

| Слой | Что выбрал | Почему |
|---|---|---|
| Backend | FastAPI 0.115 + async SQLAlchemy 2.0 + Pydantic v2 | Типы везде, mypy --strict зелёный, OpenAPI бесплатно |
| База | PostgreSQL 16 (asyncpg) + Alembic | Production-grade, не SQLite |
| Шаблоны | Jinja2 (server-side render) | Никакой гидратации, fast first paint |
| Интерактив | HTMX 2 | Свапы кусков HTML по тапу. SPA-feel без JSON-API и без bundle |
| Микро-state | Alpine.js | `x-data`, `x-show`, `x-model` — JS-фрагменты прямо в HTML |
| Стили | Tailwind CDN | НЕТ билд-процесса. `<script src="tailwindcss">` — и всё |
| Auth | Кастом на argon2 + itsdangerous | Cookie-based, без JWT |
| Email | aiosmtplib (свой SMTP) | Свой verify + password-reset |
| Логи | structlog (JSON) | Грепай по `chat_id`, `task_id` |
| Дев | uv + ruff + mypy --strict + pre-commit | Запуск зеленейший линт на каждом коммите |
| CI | GitHub Actions | `pytest -q` + lint на каждый push |
| Деплой | Cron-poll каждую минуту: git fetch + reset --hard + uvicorn restart | После `git push` прод обновляется за ~60 секунд автоматом |

Самый спорный выбор — **Tailwind CDN в продакшен**. Я знаю что это медленнее чем purged-сборка, что CSS-вес ~3MB. Но: zero-config, нет билд-процесса, нет node_modules — всё это окупает на этой стадии. Заменим на сборку через PostCSS когда станет узким местом.

**HTMX оказался шоком.** Я попробовал его две недели назад как «нишевая штучка из мема». Через сутки переписал половину UI с polling-fetch на `hx-get` + `hx-swap` и получил **более отзывчивый UI чем у React-приложений** которые видел. Все потому что нет JSON-сериализации, нет диффа виртуального DOM, нет JS-парсинга — приходит HTML-кусок, заменяет узел, всё. На мобиле особенно заметно.

## Mini App: грабли которые украли мне 6 часов

Это **обязательная часть** статьи на Хабре про Telegram Mini App, потому что граблей там больше чем кода.

### Грабли #1 — `Telegram.WebApp.themeParams` ломает твою палитру

Telegram WebApp SDK даёт `themeParams`: hex'и фона/текста/акцента из темы юзера. Логично подумать что нужно их применить к своему UI чтобы Mini App «выглядел нативно».

**Это ловушка.** Если у юзера TG в светлой теме, ты получишь `bg_color: #ffffff`. Все твои `rgba(255,255,255,0.06)` для shimmer-shimmer, `text-violet-400 bg-violet-500/15` для chips, тонкие границы `rgba(255,255,255,0.08)` — всё это рассчитано на тёмный фон и на белом смотрится как кашa.

Я первый раз слепо скопировал `themeParams` и **получил неюзабельный UI у каждого второго**. Откатил, сделал **фиксированную тёмную** тему (наш фирменный фиолетовый remained). Через сутки добавил **выбор**: тёмная / светлая / системная, светлую переписал с нуля на slate-50/100/900 + accent остался фиолет. Хранится в `localStorage.dodayTheme`, при выборе шлём `tg.setHeaderColor(...)` + `tg.setBackgroundColor(...)` чтобы chrome-бар TG поверх Mini App тоже перекрашивался. Anti-flash через inline-script в `<head>` до первого paint.

### Грабли #2 — `api.telegram.org` с IPv6 на проде = смерть бота

Бот живёт на VPS. systemd-resolved на проде отдаёт **только AAAA**-запись для `api.telegram.org`. IPv6-сеть провайдера до Telegram'a не маршрутизируется. httpx внутри python-telegram-bot честно делает `getaddrinfo` → получает IPv6 → 30s connect-timeout → polling висит → бот не отвечает на сообщения.

`curl --resolve api.telegram.org:443:149.154.167.220 https://...` работает. То есть IPv4 у Telegram есть, просто DNS отдаёт неправильное. Sudo на проде нет — `/etc/hosts` не пропатчишь.

Решение — monkey-patch `socket.getaddrinfo` и `asyncio.base_events.BaseEventLoop.getaddrinfo`:

```python
_TELEGRAM_API_IPS = ("149.154.167.220",)
_FORCED_HOSTS = {"api.telegram.org"}

def _force_ipv4_resolve() -> None:
    sync_orig = socket.getaddrinfo

    def _v4_sync(host, *args, **kwargs):
        if host in _FORCED_HOSTS:
            port = args[0] if args else kwargs.get("port", 0)
            return [(socket.AF_INET, socket.SOCK_STREAM, 6, "",
                     (ip, port)) for ip in _TELEGRAM_API_IPS]
        return sync_orig(host, *args, **kwargs)

    socket.getaddrinfo = _v4_sync

    async_orig = asyncio.base_events.BaseEventLoop.getaddrinfo

    async def _v4_async(self, host, port=0, *args, **kwargs):
        if host in _FORCED_HOSTS:
            return [(socket.AF_INET, socket.SOCK_STREAM, 6, "",
                     (ip, port)) for ip in _TELEGRAM_API_IPS]
        return await async_orig(self, host, port, *args, **kwargs)

    asyncio.base_events.BaseEventLoop.getaddrinfo = _v4_async  # type: ignore
```

### Грабли #3 — у Telegram три DC, два из них недоступны с моего хостинга

Когда я только применил patch, hardcoded были три IP: `149.154.167.220`, `149.154.166.110`, `149.154.164.220` (DC2, DC4, DC5). Бот **снова висел** — `ss -tnp` показал `SYN-SENT` к `166.110`. SYN-ACK не возвращался.

Проверил curl-ом каждый IP с прода. Из трёх работает один — DC2. На остальные asymmetric routing или провайдер режет.

httpx внутри ptb выбирает не первый IP из списка, а **случайный/round-robin** → попадал на мёртвый → connect timeout 30s → polling не двигается.

Оставил один IP. Бот ожил. Это украло у меня 6 часов вечера.

### Грабли #4 — `application.post_init = callback` в ptb v21 молча игнорируется

Я установил callback на `Application.post_init` через присваивание атрибута:

```python
application = Application.builder().token(...).build()
application.post_init = _post_init  # ← молча игнорируется в v21
```

Правильно — через builder:

```python
application = Application.builder().token(...).post_init(_post_init).build()
```

Никакого warning'a, никакого error'a. Просто `setMyCommands` не вызывался при старте. Дебагал час, пока не залез в исходники ptb.

## Анатомия: как 15-летний написал 40 000 LOC за 10 дней

Никак. Один. Это было бы враньё.

**Я писал в паре с [Claude Code](https://claude.com/claude-code)** — терминальным AI-агентом от Anthropic. Логика была такая: я придумываю фичу, описываю на словах что я хочу. Claude читает кодовую базу, предлагает дизайн, после моего «ок» — пишет код, прогоняет тесты, фиксит ошибки, коммитит.

Звучит как «всё за меня сделал AI». Это **не так** — и вот почему:

1. **Каждое решение принимаю я.** Какие фичи делать, в какой последовательности, на каком стеке, какой UX. Claude предлагает варианты — я выбираю. Когда он что-то делает не по плану, я его останавливаю и переписываю prompt.
2. **Я читаю каждый PR.** Каждый коммит — сначала diff, потом тесты, потом ручная проверка в браузере. Если что-то не нравится — переделываем.
3. **Архитектура — моя.** Я знал что хочу feature-folder структуру (`app/<feature>/{router,service,models,schemas}.py`), что хочу `mypy --strict`, что не хочу React, что не хочу подписки в бете. Это всё *мои* решения, Claude их исполняет.
4. **Грабли ловлю я.** Все 4 истории выше — это часы где я *руками* читал `ss -tnp`, смотрел `journalctl`, дёргал `getaddrinfo` через python-shell, и только потом говорил Claude'у «вот фикс, примени». AI без меня не понял бы что три IP — два мёртвые.

То есть я **не код пишу**, я **строю продукт через AI как инструмент**. Скил который реально учится — это не «писать `for i in range(n)`», а «декомпозировать задачу так чтобы AI её мог исполнить + проверять что результат правильный».

И я не понимаю людей которые до сих пор делают вид что AI-coding это «не настоящее». Через два года таких как я будет половина разработки. Тот кто *не учится* писать в паре с AI — теряет productivity-сотни процентов.

При этом меня прёт **читать** код. Я не запускаю `npm install` не зная что именно пришло. Каждый питон-импорт я хоть раз да открывал. Это разница между «вайбкодером» который копи-пастит магию из чата и **инженером** который понимает почему оно работает.

Цифры в подтверждение: я могу указать любой файл из репы и за минуту объяснить что он делает и почему так написан. Если не можешь — значит ты не разработчик, а оператор копипасты.

## Если хочешь делать что-то своё

1. **Pivot быстро.** Я через сутки понял что «только для школьников» режет аудиторию. Не жалей старого, если новое больше.
2. **Стек не должен бороться с тобой.** FastAPI + HTMX дал мне `mypy --strict` + zero JS-сборки. Это идеал для моего уровня.
3. **AI — инструмент, не замена мозгу.** Решения принимай сам. Проверяй каждую строчку. Не запускай `npm install` не зная что пришло.
4. **Открытый код — это бесплатное доверие.** Мне 15, никто меня не знает, но любой может зайти в репу и увидеть что я не вру.
5. **Беспрерывный деплой за 60 секунд** — это магия. Cron-poll + git reset --hard + uvicorn restart. 30 строк bash. Никакого Kubernetes.
