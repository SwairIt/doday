# Ralph Loop — канонический промпт авто-разработки Doday

Это промпт, которым запускается непрерывный Ralph-цикл авто-разработки Doday.
Запуск: skill `ralph-loop:ralph-loop` с этим текстом в args (без круглых скобок —
обёртка скрипта их ломает).

Останов: skill `ralph-loop:cancel-ralph`.

Обновление 2026-05-21: добавлен шаг подтверждения доставки деплоя через
`GET /version` (отдаёт git SHA прода) + самолечение при незадеплое.

---

Ралф-цикл авто-разработки Doday. Проект Doday в projectsflow, репо SwairIt/doday, рабочая директория c:/www-Yaroslav/SchoolProject, авто-деплой на getdoday.ru через cron-poll за минуту. ПОЛНАЯ АВТОНОМИЯ: пользователь даёт все права на любые идеи, главное чтобы они были. Работай непрерывно, ничего не спрашивай у пользователя, сам думай как лучше и принимай все решения сам. Не жди подтверждений. Каждую итерацию делай ПОЛНОСТЬЮ один шаг и заканчивай.

ШАГ 1 ПРОВЕРЬ ДОСКУ: pf_list_projects, найди проект Doday с gitRepoUrl SwairIt/doday, projectId fac16dd4-499a-4cba-aed2-8b9ae6bebb47, pf_list_tasks.

ШАГ 2 ЕСТЬ ОТКРЫТАЯ ЗАДАЧА todo или in_progress: возьми верхнюю, pf_get_task если есть вложения или комментарии, переведи в in_progress, реализуй ПОЛНОСТЬЮ только ДОБАВЛЯЯ не ломая существующий функционал и веб-роуты и API и архитектуру. Планка качества Doday обязательна: pre-commit зелёный - ruff format check, ruff check E F I UP B S A RUF, mypy strict app и scripts, lint_templates. Затем pytest -q зелёный. Перезапусти uvicorn по процедуре из CLAUDE.md - убей PID на 8000 через netstat, старт с reload, curl-проверь новый роут на 401 не 404. Playwright-смоук: страница грузится, ноль console errors. Скриншоты в docs/screenshots сохрани и закоммить. Commit Russian past-tense с author 112168281+SwairIt@users.noreply.github.com. Пушь через TOKEN из .env на SwairIt/doday master.

ПОДТВЕРДИ ДЕПЛОЙ (обязательно, до закрытия задачи): запомни запушенный SHA git rev-parse HEAD. Полль GET https://getdoday.ru/version каждые 10 сек до ~4 минут, пока поле sha не станет равно запушенному SHA. Когда совпало - деплой дошёл, затем smoke_test https://getdoday.ru должен быть зелёным. ЕСЛИ за 4 минуты SHA так и не совпал ИЛИ smoke красный - деплой застрял, ЧИНИ САМ: зайди по SSH на getdoday@getdoday.ru (пароль SSH_PASS из .env, без sudo), найди рабочую копию репозитория Doday на сервере, выполни git pull и перезапусти приложение, проверь лог cron-poll и сам процесс деплоя, устрани причину чтобы впредь деплоилось само. После починки снова полль /version пока не совпадёт и smoke не позеленеет. НЕ закрывай задачу пока прод не отдаёт именно запушенный SHA и smoke не зелёный. Если для починки нужны доступы которых нет - напиши об этом в комментарии задачи, но сперва попробуй всё доступное.

Затем ОБЯЗАТЕЛЬНО pf_link_commit_to_task привяжи коммит к задаче, и pf_create_task_comment добавь комментарий-отчёт что именно сделал плюс пути к скринам плюс подтверждение что прод отдаёт нужный SHA, и pf_write_kb_document worklog. Затем pf_move_task в done. Обнови PROGRESS.md.
