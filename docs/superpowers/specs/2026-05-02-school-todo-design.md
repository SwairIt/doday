# SchoolTodo — дизайн-документ

**Дата:** 2026-05-02
**Статус:** Черновик, ждёт ревью пользователя

## 1. Цель

Веб-приложение, которое автоматически вытаскивает домашние задания школьника из электронного дневника, превращает их в удобный туду-лист с чекбоксами, и помогает не забывать что и когда делать.

Школьник пользуется бесплатно навсегда. Деньги приносит платная подписка для **родителя**, который через свой кабинет видит ДЗ, прогресс и статистику ребёнка.

## 2. Целевая аудитория

- **Основной пользователь:** школьник 5-11 классов МО, авторизующийся в Школьном портале МО (authedu.mosreg.ru). На втором этапе — школьник Москвы (МЭШ).
- **Платящий клиент:** родитель этого школьника.

## 3. Не-цели (что мы НЕ делаем)

- ❌ **Авто-выполнение ДЗ нейросетью.** Изначально была такая идея, отказались — этически плохо, родитель за это не заплатит, юридические риски.
- ❌ **Не показываем оценки.** Они уже есть в дневнике, для туду не нужны.
- ❌ **Не пишем свой дневник.** Только читаем существующие.
- ❌ **Не делаем social-фичи** (рейтинги, классы, общение). Слишком много работы по модерации детского контента.
- ❌ **Не подменяем учителя.** Никаких объяснений, оценивания, проверок.

## 4. Источники данных (DiarySource абстракция)

Все источники реализуют общий интерфейс:

```python
class DiarySource(Protocol):
    name: str  # "mosreg", "mesh", ...
    def authenticate(credentials: dict) -> Session: ...
    def fetch_homework(session: Session, date_from: date, date_to: date) -> list[Homework]: ...
    def fetch_schedule(session: Session, date_from: date, date_to: date) -> list[Lesson]: ...
    def refresh_session(session: Session) -> Session: ...
```

**Приоритет реализации:**
1. **MosregSource** (authedu.mosreg.ru) — MVP, потому что школа главного разработчика на этой системе.
2. **MeshSource** (school.mos.ru) — после MVP, огромная аудитория (~1.5 млн школьников).
3. На потом: **MyschoolSource** (myschool.edu.ru), **DnevnikRuSource**, **ElzhurSource**.

В MVP реализован **только MosregSource**, но абстракция уже на месте — добавление второго источника должно занимать ~3-5 дней работы, не требуя переделки ядра.

## 5. MVP — что входит в первый релиз

**Включено:**
- Регистрация школьника по email + паролю (подтверждение email обязательно).
- Подключение Школьного портала МО через ручную вставку токена сессии (с пошаговой инструкцией и скриншотами в интерфейсе).
- Фоновая синхронизация ДЗ и расписания каждые 4 часа + ручной "Синхронизировать сейчас".
- Список ДЗ на текущую неделю и следующую неделю с чекбоксами "сделано".
- Базовое расписание уроков на день и неделю.
- Поиск по заданиям.
- Профиль пользователя (изменить пароль, отключить дневник, удалить аккаунт со всеми данными).

**Исключено из MVP (планы на потом):**
- Родительский кабинет и биллинг.
- Telegram-бот.
- MAX-бот.
- Push/email уведомления.
- Поддержка МЭШ и других дневников.
- Партнёрки.
- Госуслуги OAuth.

**Логика:** сначала проверяем что школьники вообще пользуются продуктом и синхронизация работает стабильно. Без живых пользователей родительский кабинет некому будет тестировать. После 2-4 недель боевой эксплуатации MVP — добавляем родительский кабинет и подписку.

```
                   │
┌──────────────────▼─────────────────────────────────┐
│  FastAPI приложение (uvicorn)                       │
│  ├── routers/        — HTTP-эндпоинты               │
│  ├── templates/      — Jinja2 шаблоны               │
│  ├── services/       — бизнес-логика                │
│  ├── diary_sources/  — DiarySource имплементации    │
│  ├── models/         — SQLAlchemy модели            │
│  └── auth/           — регистрация/логин            │
└────────┬──────────────────────────────┬─────────────┘
         │                              │
┌────────▼─────────┐          ┌─────────▼──────────┐
│ PostgreSQL       │          │ Redis (брокер)      │
│ (пользователи,   │          └─────────┬──────────┘
│  задания,        │                    │
│  расписание)     │          ┌─────────▼──────────┐
└──────────────────┘          │ Dramatiq workers    │
                              │ (фоновая синхро-    │
                              │  низация дневников) │
                              └─────────┬──────────┘
                                        │
                              ┌─────────▼──────────┐
                              │ authedu.mosreg.ru   │
                              │ (внешний дневник)   │
                              └────────────────────┘
```

### Модули и их ответственность

- **`routers/`** — тонкие HTTP-обработчики, валидация входа, делегирование в services.
- **`services/`** — бизнес-логика (синхронизация, отметка выполнения, поиск).
- **`diary_sources/`** — изолированный слой работы с дневниками. Каждый источник в своём подмодуле, реализует интерфейс `DiarySource`.
- **`models/`** — модели БД и pydantic-схемы.
- **`auth/`** — регистрация, логин, сессии, подтверждение email, восстановление пароля.
- **`tasks/`** — Dramatiq задачи (sync_diary, send_email).
- **`templates/`** — Jinja2-шаблоны UI.

### Поток данных синхронизации

1. Dramatiq запускает `sync_diary(user_id)` каждые 4 часа для каждого пользователя с подключённым дневником.
2. Worker берёт зашифрованный токен сессии из БД, расшифровывает в памяти.
3. Зовёт `MosregSource.fetch_homework(session, today, today + 14d)`.
4. Сравнивает результат с тем что в БД, делает diff:
   - Новые задания — создаём.
   - Изменённые — обновляем.
   - Удалённые из дневника — мягко скрываем (не удаляем — могут быть отметки "сделано").
5. Сохраняет результат, обновляет `last_sync_at`.
6. Если получили 401 от authedu.mosreg.ru — помечаем дневник как `disconnected`, шлём пользователю уведомление в интерфейсе.

## 7. Tech stack

| Слой | Технология | Почему |
|---|---|---|
| Backend | Python 3.12 + FastAPI | Сила пользователя в Python; FastAPI быстрый и понятный |
| ORM | SQLAlchemy 2.0 + Alembic | Стандарт де-факто, миграции |
| БД | PostgreSQL 16 | Reliable, любой РФ-хостинг поддерживает |
| Очереди | Dramatiq + Redis | Проще Celery, нужного функционала хватает |
| Шаблоны | Jinja2 | Серверный рендер, минимум JS |
| Frontend interactivity | HTMX 2.x | HTML-атрибуты для AJAX, не нужно учить JS |
| Frontend мелочи | Alpine.js 3 | Декларативный inline JS на крайний случай |
| Стили | Tailwind CSS 4 | Быстро, без своих CSS-файлов |
| HTTP-клиент | httpx (async) | Для запросов в дневник |
| Тесты | pytest + pytest-asyncio | Стандарт |
| Контейнеры | Docker + Compose | Воспроизводимое окружение |
| Reverse proxy | nginx | TLS, статика |
| TLS | Let's Encrypt (certbot) | Бесплатно |

**Что НЕ используем и почему:**
- React/Vue/Svelte — лишний JS, у разработчика слабый JS, для данной задачи overkill.
- Celery — Dramatiq проще и достаточен.
- MongoDB и прочий NoSQL — данные строго реляционные.
- Django — FastAPI лучше для API-first приложения.

## 7.5. UX и визуальный дизайн

Целевая аудитория — школьники 5-11 классов. Они выбирают приложения сердцем, не разумом: должно быть **красиво, быстро, приятно тыкать**, иначе откроют один раз и забудут.

### Принципы

1. **Mobile-first.** Школьник смотрит ДЗ с телефона по дороге из школы. Десктоп вторичен — но должен выглядеть прилично.
2. **Большие тапы, минимум полей ввода.** Чем меньше школьнику нужно печатать, тем выше шанс что он будет пользоваться.
3. **Микроанимации на каждое действие.** Чекбокс при отметке "сделано" не просто меняет цвет — он анимируется (галочка рисуется + лёгкие искры/конфетти при завершении задания). Без этого продукт ощущается "мертвым".
4. **Современная эстетика.** Скруглённые углы (border-radius 12-16px), мягкие тени, много воздуха, яркие акценты на градиентах. Никаких таблиц с серыми границами.
5. **Тёмная тема обязательно.** Школьники сидят в телефоне ночью. Дефолт — системная (auto), но переключатель в видимом месте.
6. **Акцентные цвета по предметам.** Математика — синий, литература — фиолетовый, физкультура — зелёный, и т.д. Делает список ДЗ визуально читаемым с одного взгляда.
7. **Шрифт.** Inter или Manrope — современные, поддерживают кириллицу, бесплатные.
8. **Без AI-аэстетики.** Не делаем "стандартный SaaS-дашборд". Цель — чтоб выглядело как продукт, который сделали с душой, а не сгенерировали по шаблону.

### Геймификация

Главная цель геймификации — **сделать рутину "проверить ДЗ" приятной**, превратить открытие приложения в маленькую дофаминовую награду. НО важная граница: не превращаемся в Duolingo с шантажом стриков и пуш-уведомлениями "ты предал свою птичку". Игровые элементы поощряют **полезные привычки** (вовремя видеть ДЗ, отмечать сделанное), а не зависимость от приложения.

**В MVP (базовая геймификация):**
- **Прогресс-бар недели на главной странице.** Большой, заметный, с плавной анимацией заполнения. Например: `7/12 задач сделано на этой неделе` + полоска заполнения с градиентом.
- **Прогресс-бар дня.** Маленький, на карточке "сегодня". Заполняется в реальном времени по мере отметок.
- **Анимация при отметке "сделано".** Галочка рисуется (SVG path animation), карточка задания плавно перемещается в "сделанные" с лёгким bounce-эффектом, конфетти-всплеск маленьким облачком.
- **Звук на отметку (тихий, опционально).** Можно отключить в настройках. По умолчанию выключен — чтоб не пугать в школе/ночью.
- **Hover/tap эффекты везде.** Каждая интерактивная карточка слегка приподнимается на hover (transform + shadow), на tap — легко "проседает".

**После MVP (продвинутая геймификация — фаза 1.5, перед родительским кабинетом):**
- **Стрики (streaks).** "5 дней подряд закрыл все задания" — счётчик с огоньком 🔥 на главной. Сброс на 0 если пропустил день. **БЕЗ шейминга** — просто факт, никаких "ты разочаровал нас".
- **XP и уровни.** Сделал задание — +10 XP. Сделал заранее (не в день сдачи) — +20 XP. Закрыл всю неделю — +100 XP бонус. Уровни 1-50 с названиями (Новичок → Прокрастинатор → Отличник → Школьный Бог).
- **Ачивки (achievements).** "Первая пятница без долгов", "Неделя без просрочки", "10 заданий по математике подряд", "Сделал ДЗ до полуночи 7 дней". Открываются → красивый поп-ап с анимацией.
- **Маскот.** Симпатичный персонаж (предложу варианты — сова/лиса/тяночка-anime/абстракт), который реагирует на твой прогресс эмоциями. Иногда говорит фразы ("Эй, английский на завтра не забыл?"). НЕ навязчиво.
- **Темы оформления.** Дефолтная + 3-5 на выбор (например: киберпанк, пастель, анимэ-розовый, минимализм). Часть как награда за уровень.

**Реализация:** анимации через CSS transitions + Tailwind animations + библиотека `motion-one` (микро, ~5кб) для сложных. Конфетти — `canvas-confetti` (~10кб). Звуки — Web Audio API, без библиотек.

**Чего НЕ делаем в геймификации:**
- ❌ НЕ делаем агрессивные пуш-уведомления "ты пропустил день, твой стрик сгорит через 2 часа!".
- ❌ НЕ делаем платные ачивки/темы (это для школьников бесплатное приложение).
- ❌ НЕ делаем рейтинги между пользователями (вызывает токсичность, надо модерировать).
- ❌ НЕ делаем "энергию" / лимиты как в free-to-play играх (мерзко).

## 8. Подключение Школьного портала МО (auth flow)

В MVP используется **ручная вставка токена сессии**:

1. Пользователь логинится в authedu.mosreg.ru в своём браузере как обычно.
2. Открывает DevTools → вкладка Network → находит любой запрос к dnevnik.mosreg.ru.
3. Копирует значение заголовка `Authorization` (или cookie сессии — выяснится при reverse-engineering API).
4. Вставляет в форму на нашем сайте.
5. Backend проверяет токен пробным запросом, шифрует (Fernet, ключ в env), сохраняет в БД.

**Безопасность хранения токенов:**
- Шифрование симметричным ключом (Fernet/AES-GCM), ключ в переменной окружения, не в коде.
- Токен расшифровывается только в памяти worker'а во время запроса в дневник.
- При удалении аккаунта — токен удаляется немедленно.

**После MVP:** мини-расширение для Chrome/Firefox, которое автоматически вытаскивает токен и отправляет в наш API через user-confirm. Расширение НЕ является основным продуктом — это helper.

## 9. Хостинг и юридическое

**Хостинг:** российский провайдер обязателен — мы храним персональные данные граждан РФ (ст. 18 152-ФЗ, требование локализации).

Кандидаты: **Selectel** (предпочтение), **Timeweb Cloud**, **Yandex Cloud**.

Минимальный VPS на старте: 2 CPU, 4 GB RAM, 40 GB SSD, ~700-1000₽/мес.

**152-ФЗ — что делаем:**
- **Регистрация в реестре операторов ПДн Роскомнадзора** — обязательна **перед публичным запуском** (когда любой человек с улицы может зарегистрироваться на сайте). На этапе разработки и закрытой беты с друзьями НЕ требуется. Форма онлайн на pd.rkn.gov.ru, занимает ~30 минут, бесплатно.
- Политика конфиденциальности на сайте, явное согласие при регистрации (чекбокс) — реализуем сразу в MVP.
- Возможность пользователя удалить все свои данные одной кнопкой — в MVP.
- Для несовершеннолетних до 14 лет — в политике явно написать, что регистрация только с согласия родителя (формально без верификации, но юридически закрывает вопрос).

**Что НЕ делаем юридически:**
- НЕ парсим МЭШ в MVP (не наш регион, у нас данных оттуда нет, не подставляемся).
- НЕ перепродаём данные третьим лицам — никогда.
- НЕ сохраняем оценки (зачем рисковать).

## 10. Roadmap после MVP

В порядке приоритета:

1. **Продвинутая геймификация (фаза 1.5)** (~1-2 недели)
   - Стрики, XP/уровни, ачивки, маскот, темы оформления.
   - Идёт первой потому что повышает retention школьников — а это база для всех следующих фаз. Без активных школьников родительский кабинет некому продавать.

2. **Родительский кабинет + биллинг** (~3-4 недели)
   - Инвайт-код от ребёнка → родитель регистрируется и линкуется.
   - Родительский интерфейс: обзор детей, статистика, прогресс.
   - Подписка ~199-299₽/мес через ЮKassa или CloudPayments.
   - Триал 7 дней.

3. **Уведомления** (~1-2 недели)
   - Email уведомления (бесплатно, через SMTP).
   - В UI — баннеры/тосты.

4. **Telegram-бот** (~2 недели)
   - Команда `/connect` линкует TG к существующему web-аккаунту.
   - Push при синхронизации, утреннее сообщение со списком ДЗ.

5. **Поддержка МЭШ** (~1-2 недели)
   - Реализация `MeshSource` через DiarySource интерфейс.
   - Главная фича для масштабирования аудитории.

6. **MAX-бот** (~1-2 недели)
   - Аналогично TG, через API мессенджера VK MAX.

7. **Партнёрки** (~1 неделя на интеграцию первого партнёра)
   - На странице задания — блок "не понимаешь тему?" с CPA-партнёром.

## 13. Открытые технические вопросы (на этапе разработки)

Эти вопросы НЕ блокируют дизайн, но решаются в ходе реализации:

1. **Точный формат токена/сессии authedu.mosreg.ru** — выясняется через DevTools и пробные запросы при имплементации `MosregSource`.
2. **Срок жизни токена** — экспериментально, скорее всего 7-30 дней. Если короткий — нужен механизм reauth-уведомлений пользователю.
3. **Rate limits authedu.mosreg.ru** — неизвестны. Закладываем backoff и не более 1 запроса в минуту на пользователя.
4. **Точная схема ДЗ в API** — выясняется на месте.

## 14. Критерии готовности MVP

MVP считается готовым к запуску когда:
- [ ] Школьник может зарегистрироваться, подтвердить email, войти.
- [ ] Школьник может вставить токен дневника, токен валидируется и сохраняется зашифрованным.
- [ ] Фоновая синхронизация работает каждые 4 часа без ошибок.
- [ ] Список ДЗ на 2 недели отображается с чекбоксами.
- [ ] Расписание на день и неделю отображается.
- [ ] Чекбокс "сделано" сохраняется в БД и не сбрасывается при синхронизации.
- [ ] Прогресс-бар недели и дня работают, заполняются плавно при отметке.
- [ ] Анимации при отметке "сделано" (галочка + конфетти) реализованы.
- [ ] Тёмная тема работает, переключатель в видимом месте.
- [ ] Mobile-first вёрстка — на телефоне всё удобно тыкается.
- [ ] Кнопка "удалить аккаунт" удаляет ВСЕ данные пользователя.
- [ ] Сайт развёрнут на РФ-хостинге, https работает.
- [ ] Зарегистрирован оператор ПДн в РКН.
- [ ] Есть политика конфиденциальности и согласие при регистрации.
- [ ] Покрытие тестами критичных сервисов (auth, sync, diary_sources) >= 60%.
- [ ] Главный разработчик в течение 7 дней пользуется приложением сам и оно не падает.
