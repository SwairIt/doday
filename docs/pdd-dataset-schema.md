# Doday ПДД — формат датасета для сидинга

Контент Doday ПДД заливается из официального набора билетов **категории АВМ**
(40 билетов × 20 вопросов). Модель пишет загрузчик (`app/pdd/seed_load.py`),
**датасет и картинки готовит и заливает юзер локально** (правило изоляции БД).

## Что нужно подготовить

1. **JSON-файл** — массив объектов-вопросов (схема ниже).
2. **Картинки** — положить в `app/static/pdd/img/`. В JSON указывается путь
   относительно `/static` (например `pdd/img/1_3.jpg`). У вопросов без картинки
   поле `image` опускается или `null`.

## Схема одного вопроса

```json
{
  "ticket": 1,                       // номер билета (1..40), int
  "position": 3,                     // позиция вопроса в билете (1..20), int
  "topic_slug": "dorozhnye-znaki",   // латиницей, kebab-case, стабильный ключ темы
  "topic_title": "Дорожные знаки",   // отображаемое имя темы (RU)
  "topic_position": 2,               // порядок темы в списке (опц., default 0)
  "text": "Разрешён ли Вам обгон?",  // текст вопроса (RU)
  "image": "pdd/img/1_3.jpg",        // путь под /static (опц.; null если нет)
  "options": [                        // варианты ответа по порядку, 2..5 шт.
    "Разрешён",
    "Запрещён",
    "Разрешён только в светлое время"
  ],
  "correct_position": 2,             // 1-based индекс верного варианта в options
  "explanation": "Знак 3.20 ..."     // официальный комментарий (RU)
}
```

### Правила

- `topic_slug` — латиница/kebab-case (идёт в URL `/pdd/tema/{slug}`). Один и тот
  же `topic_slug` в нескольких вопросах ссылается на одну тему.
- `public_slug` вопроса генерируется загрузчиком как `bilet-{ticket}-vopros-{position}`
  (идёт в URL `/pdd/vopros/{public_slug}`) — задавать вручную не нужно.
- `correct_position` — **1-based** (первый вариант = 1).
- Пара `(ticket, position)` уникальна — это «адрес» вопроса.

## Как залить (локально)

```bash
uv run alembic upgrade head          # создаст pdd_* таблицы (миграция 0048)
uv run python -m app.pdd.seed_cli path/to/dataset.json   # см. seed_load.load_dataset
```

Загрузчик **идемпотентен**: повторный запуск обновляет существующие вопросы по
`public_slug` и пересоздаёт их варианты — можно гонять сколько угодно раз.
