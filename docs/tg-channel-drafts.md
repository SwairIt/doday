# Драфты постов для TG-канала «Дневник Doday»

**Идея канала:** дневник разработки одного todo-app. Цифры, грабли, скриншоты, открытый код. Подписчики с Хабра/HN осядут тут и получат next launch'и бесплатно.

**Название:** `@doday_diary` (если занято — `@doday_log`, `@swairit_doday`, `@doday_indie`)
**Описание (240 символов):** «Дневник Doday — to-do app в Telegram. 15 лет, FastAPI + HTMX, open source. Цифры, грабли, скрины. Сайт: getdoday.ru · код: github.com/SwairIt/doday»
**Аватарка:** `bot-avatar.png` (фиолетовая D — уже сделана)

**План публикации:** 5 ретроактивных постов с правильными back-dates (TG позволяет отредактировать дату, но скорее просто публикуй их в один день с подписями «13 мая → α завершён…»), затем 2 forward-looking для дня публикации Хабр-статьи.

## Пост 3 — β: UI redesign (13 мая, вечер)

```
13 мая, вечер. После чистки — редизайн UI.

Что было: sidebar из 9 пунктов, каждая задача рендерилась в три
строки с 8 чипсами, тёмная тема насильно (мини-апп). Шумно.

Что стало:
✓ Light тема по умолчанию — для read-heavy todo-UI белый фон
  легче. Тёмная — toggle на /me.
✓ Sidebar: 4 главных пункта (Inbox / Сегодня / Ближайшие /
  Календарь) + проекты + collapsible «Ещё» (статистика, лейблы,
  активность, корзина — нужны редко, не должны мозолить глаза).
✓ Task row — одна строка: ○ + заголовок + первый лейбл + дата +
  ⋯. Всё остальное (описание, recurrence, subtask-progress) —
  в detail-панели по тапу.

Принцип: чем меньше отвлекающих элементов в основном flow,
тем чище мозг.

Через час после деплоя нашёл что повторял один и тот же
HTTP-запрос на /api/projects/sidebar-counts ДВАЖДЫ на каждой
загрузке (Alpine scope conflict). Поправил, теперь один.
Эта мелочь — каждый день мог бы стоить 10к лишних запросов,
если бы кто-то реально активно юзал.
```

## Пост 4 — γ: комментарии (14 мая)

```
14 мая. Добавил комментарии к задачам в Mini App.

В web они уже были, в мобильном чате — нет. Доделал accordion
в bottom-sheet: тапаешь задачу → раскрываешь «💬 Комментарии» →
видишь список, добавляешь свой, удаляешь свой.

Lazy-fetch на первое раскрытие, никакой подгрузки заранее.
Подзадачи получают комментарии бесплатно — в БД они сидят в той
же таблице tasks, FK на comments.task_id одинаков.

Заодно убрал desktop hover-strip из task_row (3 кнопки которые
видны только при наведении мышью). Теперь всё через ⋯ — single
entry point.

Урок: фича может не существовать в UI даже когда бэкенд готов.
И никто этого не замечает, потому что нет UX-теста на каждом
устройстве. Когда добавил в Mini App, ушло 2 часа. Backend был
готов ещё на прошлой неделе.
```

## Пост 5 — δ: командная работа (14 мая, ночь)

```
14 мая, ночь. Закрыл самую большую фазу — shared projects.

Теперь Doday это не одиночный todo, а **командный**. Создаёшь
проект → жмёшь «Поделиться» → вводишь email → твоему
напарнику приходит письмо со ссылкой → он логинится → видит твой
проект в sidebar → может добавлять/комментировать/закрывать
задачи. Задачу можно назначить на конкретного человека.

Цифры:
→ 2 новые таблицы (project_members, project_invitations)
→ 1 миграция + backfill (каждый существующий проект
  автоматически получил owner-row)
→ 7 новых API endpoints
→ ~30 новых тестов

Самая трудная часть — permission-слой. Доступ к задаче/проекту
сейчас проверяется через `is_member(user, project_id)`, не через
`task.user_id == me`. Это значит код в 40 routers ничего не знает
про членство — security сидит в 3 service-функциях которые они
все и так зовут. Если бы не центральная точка, пришлось бы
переписывать каждый endpoint вручную.

Final security-review нашёл 0 data leaks. Functional gaps
(где member получал слишком мало доступа) — закрыл follow-up'ом.

Текущее состояние: 650 тестов зелёные, smoke 23/23 на проде,
миграция 0030 применена. Spec четырёх фаз закрыт полностью.
```

## Tips по публикации

- **Не публикуй все 7 за раз** — алгоритм TG увидит спам. Распредели на 2-3 дня.
- **Реальные числа** в пост 6/7 — заполни перед публикацией.
- **Скриншоты** добавь в посты 3, 4, 5 — пост с медиа в 5-10 раз больше CTR.
- **Хэштеги** не работают в каналах. Не трать символы на них.
- **Ссылки внутри текста** (`[заголовок](url)`) — Markdown работает в TG.
- **Один пост в день** в самом начале — выглядит как «живой канал», не как накрутка.
- **Перепост в свой личный TG** (@SwairIt) — первый пост канала. Друзья подпишутся, дают начальные 20-50 подписчиков чтобы канал не выглядел пустым.
