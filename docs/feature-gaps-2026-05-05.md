# Doday — что ещё не доделано (аудит 2026-05-05)

Прошёлся как реальный пользователь и сравнил с тем, что есть в индустриальных
todo-приложениях (Todoist, TickTick, Things, Microsoft To Do, Asana, Linear).
Цитат и UI-копий конкретных продуктов не привожу — только жанровые паттерны.

Помечено: ✅ done · 🔨 implementing now · 💡 backlog · 🚫 не делаем (out of scope).

---

## 🗓️ Календарь — главная боль

Текущая реализация: 7×6 grid, в ячейке до 3 задач + «+N ещё» как текст
без действия. На загруженном дне ничего реально не разглядеть и не открыть.

| # | Идея | Статус |
|---|---|---|
| C1 | **Клик по дню → модалка-детализация** со списком ВСЕХ задач на этот день, с кнопкой «+ добавить на этот день» | 🔨 |
| C2 | **Week-view** — альтернатива month-view, 7 колонок (Пн-Вс), каждая колонка — вертикальный список с inline-редактированием | 🔨 |
| C3 | **«+N ещё» делает то, что обещает** — кликабельное, разворачивает список или открывает C1 | 🔨 |
| C4 | **Dense-mode** — переключатель «компактно»: вместо текста задач — точки разных цветов (одна точка = одна задача, цвет = проект). Тогда даже 30 задач на день видно одним взглядом | 🔨 |
| C5 | Drag-resize задачи на календаре (потянул правый край → сдвинул дедлайн) | 💡 |
| C6 | Heatmap-режим — насыщенность ячейки = кол-во задач (как GitHub-contributions) | 💡 |
| C7 | Mini-calendar в сайдбаре с маркерами загруженных дней | 💡 |
| C8 | Drag задачи прямо в календаре между днями уже работает на /upcoming, но НЕ работает на /calendar — починить | 💡 |
| C9 | «Перейти к дате» — date-picker в шапке календаря | 💡 |
| C10 | Фильтр по проекту в календаре (показать только задачи проекта X) | 💡 |
| C11 | Печать недельной/дневной повестки | 💡 |

## План на этот батч (приоритет: пользователь явно ткнул в календарь)

1. **C1** — modal-детализация дня (клик по cell → попап со списком всех задач, кнопка «+ добавить на эту дату»)
2. **C3** — кликабельный «+N ещё» (тот же modal что C1)
3. **C2** — week-view (`?view=week` в /calendar) c day-колонками
4. **C4** — переключатель «список / точки», dense-mode на месяц
5. **L1+K1** — j/k-навигация по списку задач (фокус-маркер + Enter открывает detail)
6. **L2** — shift-click multi-select для bulk-bar
7. **K3** — `t`/`T`/`w` для быстрой смены даты выбранной задачи
8. **K5** — `1`/`2`/`3`/`4` для приоритетов выбранной
