Quick walkthrough notes, things worth doing in subsequent passes. **Tonight's
delivery** stayed narrow to ship cleanly: auto-sync of school portal +
streak chip on /today. Below are the candidate next moves, ranked by impact ÷
risk.

## High-value, low-risk follow-ups (next pass)

1. **Task-completion micro-animation** — when checkbox is clicked, the task
   row briefly scales-down + fades the title with a strikethrough sweep. Pure
   CSS, ~10 lines. Adds joy without any backend.
2. **Streak warning** — if `today_done == 0` and streak > 0, show a soft
   "🔥 не сорви серию — сделай хоть одну сегодня". Pure server-side
   conditional in `today.html`.
3. **Onboarding nudge** for new users: if `created_at < 24h ago` and tasks
   < 3, show a small "💡 первые шаги" card linking to /help/getting-started.
   Already have `onboarding_card.html` — verify it covers this.
4. **Empty state for /app/projects** — when zero projects, show a friendly
   landing with one-click "Создать первый проект".
5. **Project graph view** (user explicitly asked: «где графы?») — the
   pre-phase-α task-link graph was removed with its schema. Cheap revival:
   visualize the subtask tree of a project as a force-directed graph using
   `cytoscape.js` via CDN. New route `/app/projects/{slug}/graph`. No new
   schema (uses existing `parent_task_id`).
6. **Quick-add NLP polish** — already parses `завтра`, `п1`, `#проект`.
   Extend with `в 18:00` time parsing.
7. **Mobile bottom-bar polish** — verify the topbar collapses to a 4-tab
   bottom-nav on mobile; if not, add it (today/inbox/calendar/me).

## Things I deliberately did NOT bring back (per [[project_simplify_direction]])

XP/levels system, the 16-achievement system, mood log, habits, time-tracking,
company/standup, user-templates. These are big modules deleted in phase α.
The principle from the 2026-05-23 partial reversal: surface existing
motivational signals (already done), add small joyful micro-interactions —
but don't resurrect heavy removed modules from scratch.
